The payment rail for
agent-native AI access.

Tab lets any human or autonomous agent pay per request, on-chain, for premium AI - no subscription, no card, no account. Fund a balance, spend it down to zero, and hit a wall it can't cross. We proved the rail with a paid API. The destination is direct, pay-as-you-go access to Claude.

HTTP 402 · x402 · Casper · CEP-18 settlement · one signed authorization per call

The shift

From a primitive to a product

The primitive is done and running: pay-per-call settlement on-chain, gating a real API. Text-to-speech was the proof. The same rail, pointed at a frontier model, becomes something people actually want - metered access to Claude.

What we built

Pay-per-call → Deepgram TTS

An agent signs an x402 authorization, the facilitator settles a CEP-18 transfer on Casper, and the rail returns spoken audio. Real money, real API, hard spend cap.

→
Where it's going

Pay-per-message → Claude

The same rail in front of the Claude Messages API. Chat with Claude and pay per message from an on-chain balance - for people and for agents that pay their own way.

Roadmap

Four phases to agent-native Claude access

Each phase ships something usable. We don't wait for the smart-contract escrow to put Claude behind the rail - we harden the guarantees as we scale.

01

The primitive

Shipped

Prove that an agent can pay for a real API, per call, on-chain, with a cap it cannot exceed.

  • x402 paywall (HTTP 402 + price) in front of a live API
  • Own CEP-18 token on Casper testnet; facilitator verifies signatures & settles transfers, pays gas
  • Per-call settlement - one signed authorization, one on-chain transfer, real spoken audio back
  • Live dashboard, house-funded demo, and a global on-chain settlement feed
  • Single-account treasury: holds supply, pays gas, signs settlements
02

Claude, metered

Next up

Point the rail at Claude. Pay-per-message chat, streamed back, priced by tokens.

  • New /v1/chat endpoint gating the Claude Messages API Anthropic
  • Token-accurate pricing - cost scales with input + output tokens, quoted in the 402
  • Streaming responses (SSE) with pre-authorization, settled on completion
  • "Chat with Claude, pay per message" web UI - the demo becomes the product
  • Model choice: Opus / Sonnet / Haiku, each with its own per-token price
03

Developer platform

Planned

Let anyone build on the rail: keys, balances, metering, a drop-in SDK.

  • API keys tied to on-chain funded balances
  • Real-time usage metering - spend, remaining balance, per-call history
  • Per-key spend caps & rate limits set by the developer
  • Drop-in fetch wrapper SDK - any app pays automatically on 402
  • Provider-agnostic gateway: many APIs behind one rail and one balance
04

Hard guarantees & scale

Vision

Make the spend cap a contract-enforced guarantee, then go wide.

  • Odra budget-escrow contract - cumulative ceiling & per-merchant sub-limits the agent physically cannot exceed
  • Mainnet launch
  • Fiat on-ramp - top up a balance with a card, for users who never touch crypto
  • Multi-provider marketplace - Claude alongside other premium APIs, one rail
  • Agent-to-agent payments - agents settling each other's Tabs autonomously
