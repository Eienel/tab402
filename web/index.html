Give your agent a budget it can't blow.

Autonomous agents hold crypto but the world's APIs run on cards. Casper Agent Rail lets an agent pay for real APIs per request from an on-chain balance — with a hard spending cap it physically cannot exceed.

The gap

x402 lets agents pay crypto-native services. It does nothing for the 99% of APIs that only take a card — and nothing to stop an agent from draining your account overnight.

Agents can't hold a Visa

A non-human economic actor has a wallet, not a credit card. The fiat world won't onboard it.

No spend ceiling

Hand an agent an API key and it can burn $10k while you sleep. Nobody ships that to production.

No verifiable metering

You trust the provider's invoice. There's no on-chain proof you were charged for exactly what you used.

How it works

01

Agent calls the rail

Points at our endpoint instead of the provider, holding a Casper key + token budget.

02

402 + price

The rail answers 402 Payment Required with a per-call price in our CEP-18 token.

03

Sign & settle

The agent signs an x402 authorization; the facilitator settles it on Casper. Instant finality (Zug).

04

Fulfilled

The rail forwards to the real API (Deepgram) and returns the result — with an on-chain receipt.

The moat: spend-bounded access

The agent's on-chain token balance is its cap. Run it dry and the next call simply can't settle. Overspend is impossible, not just discouraged.

Why Casper

Zug's deterministic instant finality makes per-call on-chain settlement viable — impossible on probabilistic chains. Every API call is one real transaction.
